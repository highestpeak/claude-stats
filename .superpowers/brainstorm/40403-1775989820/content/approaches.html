三种架构方案

核心问题：如何组织新增的 Token 详情、项目分布、Prompt 历史这三大板块

A

单页滚动 + 折叠区块

保持现有单页结构，新增内容作为可折叠 section 插入。无导航切换。

优点

  • 改动最小
  • 一屏滚到底截图

缺点

  • 页面极长，Prompt 列表体验差
  • 项目分布和历史挤在一起
B

⭐ 多页 + 顶部 Tab 导航（推荐）

4 个独立页面：Overview / Tokens / Projects / Prompts。顶部 tab 切换，每页专注一个主题。

优点

  • 每页专注，截图干净
  • Prompts 页有足够空间做搜索/过滤
  • 扩展性强，以后加新页面很容易

缺点

  • 需要加路由（Next.js 已有，成本低）
  • 分享时需要分别截图各页
C

侧边栏导航 + 主内容区

左侧固定导航栏，右侧内容区切换页面。类似 VSCode / Linear 风格。

优点

  • 导航始终可见，专业感强
  • 适合未来功能增多

缺点

  • 侧边栏占宽度，图表空间减少
  • 移动端体验需要额外处理
  • 截图时侧边栏总在
